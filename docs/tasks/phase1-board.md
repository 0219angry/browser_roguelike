# Phase 1: 盤面成立

企画書 §17.2 Phase 1 に対応。ゲームとして動く最小単位を作る。

## タスク一覧

### 1-1. グリッドレンダリング
- 8x8 のタイルグリッドをブラウザ上に表示する
- タイル種別: 床 / 壁 / 危険地形 / 階段 を色分け表示
- プレイヤー・敵のアイコンをグリッド上に重ねて表示
- 参照: §3.1（フロア基本仕様）、§14.1（Floor, Tile エンティティ）

### 1-2. プレイヤー移動
- 隣接1マスへの移動（上下左右）を実装（コスト 1 AP）
- 壁・敵への移動を禁止
- AP は毎ターン全回復（初期 2 AP）
- 参照: §4.1（基本ルール）、§4.2（行動表）

### 1-3. 近接攻撃
- 隣接した敵へのダメージ処理（初期攻撃力 3）
- 攻撃コスト 1 AP
- 攻撃後に敵 HP を更新し、0 以下なら除去
- 参照: §4.2、§4.3、§19（初期チューニング）

### 1-4. 敵ターン処理
- プレイヤー行動終了後に敵フェーズへ移行
- 敵は intent（予告行動）を実行する（Phase 2 で予告表示を追加）
- 複数敵の順序処理を定義（敵 ID 順など）
- 参照: §4.1

### 1-5. HP 処理・死亡判定
- プレイヤー被ダメージ処理（防御軽減 2 を考慮）
- プレイヤー HP が 0 以下でゲームオーバー画面へ遷移
- 敵 HP が 0 以下で盤面から除去
- 参照: §4.3（基本ステータス）

### 1-6. ターン管理
- プレイヤーターン → 敵ターン のフロー制御
- AP が 0 または手動ターン終了で敵フェーズに移行
- ターン数カウンター（危険度上昇の基礎、Phase 3 で使用）
- 参照: §4.1

## 完了条件
- 8x8 盤面でプレイヤーが移動・攻撃でき、敵が反応し、HP が正常に増減する
