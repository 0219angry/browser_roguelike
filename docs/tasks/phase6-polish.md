# Phase 6: 調整・仕上げ

企画書 §17.2 Phase 6 に対応。KPI を見ながら数値・UI・体験を整える。

## タスク一覧

### 6-1. 報酬逓減の調整
- 経験値効率カーブの実測値を確認し調整
- 増援狩りが正解にならない状態を確認
- 参照: §5.7、§16.3（悪い状態チェック）

### 6-2. 危険度カーブ調整
- 各フェーズの体感圧力を確認
- 1フロア平均滞在ターン 5〜9 ターンに収まるか測定
- 危険度6以上到達率 50〜70% を目標に調整
- 危険度9以上到達率 15〜30% を目標に調整
- 参照: §16.1

### 6-3. KPI チェック
以下を実プレイで確認する:
- 平均ラン時間: 25〜45分
- 即降り率: 20% 未満
- 死因の上位が「撤退遅れ」であること
- プレイヤーごとに撤退タイミングが分かれること
- 参照: §16.1〜16.3

### 6-4. UI 整理
必須表示の確認（§13.1）:
- プレイヤー HP
- 危険度ゲージ
- 階段位置
- 敵の予告表示
- 危険地形
- 所持消耗品
- 現在の遺物アイコン
- 装備3枠
- アクティブスキル2枠
- 情報優先度（次の危険 → 逃げ先 → 行動選択肢 → ビルド変化）に合わせてレイアウト調整
- 参照: §13.1〜13.2

### 6-5. 演出の最小化・高品質化
- 大きなアニメーションを排除
- 被弾 / 押し出し / 予告更新を短く明瞭に
- ログ表示の位置・量を整理
- 参照: §13.3

### 6-6. 5点チェックリスト（企画の芯）
リリース前に必ず確認する:
- [ ] 長居は本当に魅力的か
- [ ] でも長居し続けるのは本当に損か
- [ ] 即降りにも本当に価値があるか
- [ ] ビルド要素はその判断を本当に揺らしているか
- [ ] ショップや端末が主題を食っていないか
- 参照: §18

## 完了条件
- 全KPIが目標範囲内で、5点チェックリストが全てクリアされた状態
