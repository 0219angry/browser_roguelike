# Phase 2: 読み合い成立

企画書 §17.2 Phase 2 に対応。「見えている脅威にどう応じるか」を成立させる。

## タスク一覧

### 2-1. 敵行動予告表示
- ターン開始時に各敵の次行動を盤面上に表示（アイコン or 矢印）
- 予告種別: 近接攻撃 / 直線射撃 / 範囲爆破 / 移動後攻撃 / 増援召喚 / 危険地形生成
- 予告は敵フェーズ実行前に必ず視認できる状態にする
- 参照: §4.5

### 2-2. 押し出し / 引き寄せ処理
- ノックバック: 対象を指定方向へ1マス移動させる
- 壁衝突時に追加ダメージ（スキル・遺物との連携基盤）
- 引き寄せ: 対象を1マス手前に移動させる
- 同一マス重複禁止ルールを考慮
- 参照: §4.4

### 2-3. 危険地形
- 危険マス（ダメージタイル）の初期配置（0〜3マス）
- プレイヤー・敵がそのマスに侵入 or 滞在時にダメージ
- 危険マスの視覚的区別（色 / アイコン）
- 参照: §3.1、§11.2

### 2-4. 基本敵 4 種実装
以下を最低限実装する（§10.2 より）:
- **Striker**: 近接突撃（1マス移動後に攻撃）
- **Gunner**: 直線3マス射撃
- **Pusher**: 近接攻撃 + ノックバック
- **Seeder**: 危険マス設置
- 各敵の intent ロジックを定義し、フロア生成時にランダム配置できる状態にする
- 参照: §10.1、§10.2

### 2-5. 防御アクション
- 防御（コスト 1 AP）: 次に受けるダメージを 2 軽減
- 待機・防御に派生効果の基盤を用意（遺物・スキル改造で拡張できる構造）
- 参照: §4.2、§4.4

## 完了条件
- 敵の予告を見て位置取りを判断でき、ノックバックや危険地形が戦術に影響する状態
